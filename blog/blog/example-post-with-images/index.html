Using Images in Blog Posts

This post demonstrates how to add both preview images and inline images to your blog posts.

Preview Images

Preview images are set in the frontmatter using the image field. They appear:

  1. In the blog listing as a preview thumbnail
  2. At the top of the post as a splash/hero image

The preview image for this post is defined in the frontmatter as:

image: '/blog/assets/sample-preview.jpg'
imageCaption: 'This is a preview image that appears at the top of the post and in the blog listing'

Image Organization

Store your blog images in /blog-src/assets/ and they'll be automatically copied to /blog/assets/ in the built site.

Recommended structure:

blog-src/
  assets/
    post-images/
      my-post-name/
        hero.jpg
        diagram.png
        screenshot.jpg
    shared/
      author-avatar.jpg
      default-preview.jpg

Best Practices

  1. Preview images: Use landscape format (16:9 or 2:1 ratio) for best results
  2. File sizes: Optimize images for web (aim for <500KB for previews, <200KB for inline)
  3. Alt text: Always provide descriptive alt text for accessibility
  4. File naming: Use descriptive, URL-friendly names (no spaces, lowercase)

Path Examples

  • Absolute path: /blog/assets/my-image.jpg (recommended)
  • Relative path: ../assets/my-image.jpg (works but less reliable)
  • External URLs: https://example.com/image.jpg (works but not recommended for performance)
